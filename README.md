# 🎧 Audio Junkie

## 📌 Description

**Audio Junkie** est un mini lecteur audio web développé en HTML, CSS et JavaScript.\
Il permet de lire des fichiers audio locaux avec une interface simple, moderne et responsive.

---

## 🚀 Fonctionnalités

- ▶️ Lecture / pause
- ⏭ Passage à la piste suivante
- ⏮ Retour à la piste précédente
- 🔊 Contrôle du volume (avec icône dynamique)
- 📃 Playlist interactive
- 🎵 Lecture automatique de la piste suivante
- ⏱ Barre de progression cliquable
- ⏰ Affichage du temps (actuel / durée totale)
- 🖼️ Affichage de la couverture (album)
- 📱 Interface responsive (PC et mobile)

---

## 🛠️ Technologies utilisées

- **HTML5** → structure de la page
- **CSS3** → design et responsive
- **JavaScript (Vanilla)** → logique et interactions (Audio API, DOM)

## ▶️ Utilisation

- Cliquer sur ▶️ pour lancer la musique
- Utiliser ⏭ / ⏮ pour naviguer entre les pistes
- Ajuster le volume avec le slider
- Cliquer sur la barre de progression pour avancer dans la musique
- Sélectionner une piste directement depuis la playlist

## 🆕 Nouveautés explorées

Durant ce projet, j’ai découvert et approfondi plusieurs concepts importants :

- Utilisation de l’API Audio en JavaScript
- Manipulation du DOM pour rendre l’interface interactive
- Chargement dynamique de données via un fichier JSON
- Gestion d’événements (play, pause, ended, timeupdate, etc.)
- Création d’une interface responsive avec CSS (media queries)

---

## ⚠️ Difficultés rencontrées

Plusieurs difficultés ont été rencontrées durant le développement :

- Problèmes de lecture audio (fichiers non reconnus ou mauvais chemins)
- Synchronisation entre l’état du bouton (play/pause) et l’audio
- Gestion de la barre de progression et du temps
- Problèmes d’affichage responsive sur mobile
- Erreurs lors du push sur GitHub (fichiers lourds, timeout)

---

## ✅ Solutions apportées

Pour résoudre ces problèmes, plusieurs approches ont été utilisées :

- Vérification et correction des chemins des fichiers audio
- Utilisation des événements audio (play, pause, ended) pour synchroniser l’interface
- Ajout de conditions pour éviter les erreurs (ex: durée non chargée)
- Utilisation des media queries pour adapter le design mobile
- Recherche en ligne (documentation, forums) et tests progressifs
- Optimisation des fichiers et nouvelles tentatives de push sur GitHub
